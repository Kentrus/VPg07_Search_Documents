# VPg07. Поиск по своим данным (Haystack + Docling)

Домашнее задание к уроку VPg07 курса "Профессия вайб-кодер" (Zerocoder), модуль 7 "Работа с памятью LLM".

## Описание

Персональный ассистент для MAX messenger с возможностью **поиска по загруженным документам**. Построен на модульной архитектуре с двумя пайплайнами.

Бот умеет:
- **Анализировать документы** (PDF, DOCX) через Docling и сохранять в Pinecone
- **Отвечать на вопросы** по содержимому загруженных файлов
- **Выдавать резюме** после загрузки документа
- **Запоминать контекст** разговоров
- **Вызывать инструменты** — факты о кошках, картинки собак, погода

## Два пайплайна

| Пайплайн | Направление | Когда работает | Что делает |
|---|---|---|---|
| **Ingestion** | Данные → В базу | Пользователь загрузил файл | Docling парсит → чанки → эмбеддинги → Pinecone |
| **Generation** | Данные → Из базы | Пользователь задал вопрос | Поиск в Pinecone → контекст → Agent → ответ |

## Параметры чанкинга

Документы разбиваются на **большие чанки** для экономии вызовов OpenAI Embeddings API:

| Параметр | Значение | Комментарий |
|---|---|---|
| `CHUNK_SIZE` | 2000 символов (~300 слов) | Крупные чанки — меньше запросов к API |
| `CHUNK_OVERLAP` | 200 символов | Перекрытие, чтобы не терять смысл на границах |

Для продакшна рекомендуется уменьшить до 500-1000 символов — точнее поиск, но дороже.

## Стек

- **Python 3.12+**
- **haystack-ai 2.27** — фреймворк для AI-агентов
- **docling 2.89** — парсинг PDF, DOCX и других форматов
- **maxapi** — MAX messenger SDK (вместо Telegram)
- **OpenAI API** — через прокси api.proxyapi.ru (эмбеддинги + LLM)
- **Pinecone** — векторная база для памяти и документов
